\chapter{Удиртгал}
\label{ch:introduction}

\section{Үндэслэл, ач холбогдол}

Гар утасны хэрэглээ огцом өссөнтэй холбоотойгоор өдөр тутмын тэмдэглэл, сэтгэл хөдлөл хянах, өөрийгөө ажиглах төрлийн аппликейшний хэрэгцээ нэмэгдэж байна. Гэвч олон хэрэглээ одоог хүртэл энгийн текст хадгалалт дээр төвлөрсөн, аюулгүй байдал сул, хэрэглэгчийн оролцоог тогтвортой хадгалах интерактив боломж хязгаарлагдмал хэвээр байна.

Олон платформт хөгжүүлэлтийн орчин болох Flutter нь нэг кодын сангаас Android болон iOS орчинд аппликейшн бүтээх боломж олгож, хөгжүүлэлтийн хугацаа ба зардлыг бууруулдаг \cite{flutter_docs}. Нөгөө талаас Node.js болон Express дээр суурилсан сервер талын хэсэг (RESTful backend) нь хөнгөн, өргөтгөх боломжтой, харагдах хэсэг (frontend)-тэй хурдан уялдах давуу талтай \cite{nodejs_docs}\cite{express_docs}. Өгөгдөл хадгалалтын түвшинд MongoDB ашигласнаар тэмдэглэл, хэрэглэгчийн профайл зэрэг бүтэц нь өөрчлөгдөх өгөгдлийг уян хатан загварчлах боломжтой \cite{mongodb_docs}.

Хиймэл оюуны үйлчилгээний хувьд OpenAI-ийн хэрэглээний програмчлалын интерфейс (API)-д суурилсан зураг үүсгэх боломж нэмэгдэх нь энгийн тэмдэглэл хөтлөх аппликейшнийг хэрэглэгчтэй харилцах бүтээлч платформ болгон өргөтгөж байна \cite{openai_images}. Энэ нь хэрэглэгчийн оролцоог нэмэгдүүлж, аппликейшнийг илүү сонирхолтой, интерактив хэрэглээтэй болгохын зэрэгцээ бусад ижил төрлийн үйлчилгээнээс ялгарах өрсөлдөх давуу талыг бүрдүүлэхэд чухал ач холбогдолтой.

Иймд энэхүү судалгаанд Flutter дээрх харагдах хэсэг (frontend) болон Node.js дээрх сервер талын хэсэг (backend)-ийг ашигласан, JWT хамгаалалттай, хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) функцтэй ``AI Notebook'' системийг инженерчлэлийн түвшинд боловсруулж, практикт ашиглах боломжийг үнэлж байна.

\section{Зорилго, зорилт}

\subsection{Судалгааны зорилго}

Энэхүү судалгааны ажлын гол зорилго нь харагдах хэсэг (frontend) болон Node.js/Express дээрх сервер талын хэсэг (backend)-д тулгуурласан, хэрэглэгчийн баталгаажуулалт, тэмдэглэлийн менежмент, хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) боломж бүхий ухаалаг тэмдэглэлийн гар утасны апп зохион бүтээж, хөгжүүлэн, түүний ажиллагаа, үр нөлөөг туршилтаар үнэлэхэд оршино.

\subsection{Судалгааны зорилтууд}

Дээрх зорилгод хүрэхийн тулд дараах зорилтуудыг дэвшүүлсэн:

\begin{enumerate}
    \item Ухаалаг тэмдэглэлийн системийг ашигласнаар хэрэглэгч өдөр тутмын тэмдэглэл, төлөвлөгөөгөө илүү эмх цэгцтэй, үр дүнтэй удирдах боломжтой болно.
    \item OpenAI API ашиглан хэрэглэгчийн оруулсан текст болон хүсэлтэд үндэслэн хиймэл оюун ухаанаар стикер (sticker) автоматаар үүсч, тэмдэглэлийн агуулгыг илүү ойлгомжтой болгоно.
    \item Үүсгэсэн стикерүүдийг календарьтай уялдуулан тухайн өдрүүд дээр харагдуулах боломжтой болж, хэрэглэгч календарийн харагдацаар өдөр бүрийн тэмдэглэл болон стикерээ нэг дор хянах боломжтой болно.
    \item Календарийн тодорхой өдөр дээр дарснаар тухайн өдөртэй холбоотой хийх ажлууд (tasks) болон үүсгэсэн стикерүүдийг дэлгэрэнгүй харах, удирдах боломж бүрдэнэ.
    \item Стикер бүрт сануулах тохиргоо (reminder) тохируулснаар хэрэглэгчид шууд мэдэгдэл (push notification) хэлбэрээр сануулга ирж, хийх ажлаа цаг тухайд нь сануулах боломжтой болно.
\end{enumerate}

\section{Судалгааны хамрах хүрээ}

Энэхүү судалгаанд дараах хүрээг баримталсан:

Системийг хөгжүүлэхдээ мобайл талд Flutter (Dart), сервер талд Node.js, Express болон MongoDB технологийг ашигласан бөгөөд AI функцэд OpenAI-ийн зураг үүсгэх API-г нэгтгэсэн. Мөн системийн үнэлгээг функцийн туршилт, өгөгдлийн урсгалын баталгаажуулалт болон дарааллын диаграммын (sequence diagram) шинжилгээнд тулгуурлан гүйцэтгэсэн.

Судалгааны гадна үлдээсэн хүрээ:

Системийг үйлдвэрлэлийн орчинд (production environment) бүрэн байршуулж, түлхэх мэдэгдлийн үйлчилгээ (push notification service)-г идэвхжүүлсэн бөгөөд үүлэн орчны автоматаар нөөц нэмэх механизм (cloud auto-scaling) болон тасралтгүй нэгтгэл-байршуулалтын шугам (CI/CD pipeline)-ыг багтаасан DevOps шийдлийг хэрэгжүүлсэн. Мөн аппликейшнийг түгээлтийн дэлгүүрт нийтлэх үед бизнес, хууль эрх зүй болон төлбөрийн интеграцийн шаардлагуудыг хангаж ажилласан.

\section{Судалгааны ажлын бүтэц}

Энэхүү төгсөлтийн судалгааны ажил дараах бүтэцтэйгээр зохион байгуулагдсан.

\begin{enumerate}
    \item Нэгдүгээр бүлэгт судалгааны үндэслэл, зорилго, зорилт, хамрах хүрээг тодорхойлсон.
    \item Хоёрдугаар бүлэгт сэдэвтэй холбоотой онолын суурь, хэрэглэгдэж буй технологийн өнөөгийн түвшинг тоймлосон.
    \item Гуравдугаар бүлэгт системийн арга зүй, өгөгдлийн урсгалын түвшинчилсэн задлан шинжилгээ, дарааллын диаграммуудыг (sequence diagrams) тайлбарласан.
    \item Дөрөвдүгээр бүлэгт хэрэгжилтийн үр дүн, интеграцийн туршилт, баталгаажуулалтын дүнг нэгтгэн харуулсан.
    \item Тавдугаар бүлэгт судалгааны ерөнхий дүгнэлт, хязгаарлалт, цаашдын хөгжүүлэлтийн чиглэлийг тодорхойлсон.
\end{enumerate}
