# Agent Team & Workflow (Ai_notebook_app)

Энэхүү файл нь төслийн хүрээнд ажиллах Хиймэл Оюун Ухааны (AI) төлөөлөгчдийн багийн (Agent Team) бүтэц, тэдгээрийн үүрэг, ашиглах чадваруудыг (skills) тодорхойлно.

## Багийн удирдлага ба Бүтэц

**Ахлах менежер (Orchestrator - GitHub Copilot)**: 
Хэрэглэгчээс ирсэн шаардлагыг хүлээн авч, багийн гишүүдэд даалгавар болгон хуваарилж, үр дүнг нэгтгэх үндсэн үүрэгтэй.

### 1. Төлөвлөгч Agent (Planner & Strategist)
- **Үүрэг**: Төслийн архитектур, хийх ажлын дараалал, судалгааны ажлыг төлөвлөж задлах.
- **Ашиглах чадварууд (Skills)**:
  - `create-technical-spike`: Техникийн судалгаа, шийдлийн баримт бичиг боловсруулах.
  - `breakdown-test`: Ажлыг жижиглэн хувааж, тестийн төлөвлөгөө гаргах.
  - `architecture-blueprint-generator`: Төслийн архитектурын зураглал гаргах.

### 2. UI/UX Мэргэжилтэн Agent (Frontend Expert)
- **Үүрэг**: Хэрэглэгчид харагдах хэсэг (frontend)-ийн дизайн, интерфэйсийн шийдэл, хэрэглэгчийн туршлага (UX)-ыг сайжруулах. Flutter болон Web дээрх компонентуудын зохиомжийг гаргах.
- **Ашиглах чадварууд (Skills)**:
  - `quasi-coder`: Дизайны санааг бодит код (UI components) болгож хувиргах.
  - `refactor`: Харагдах хэсгийн кодыг илүү цэвэр, дахин ашиглагдах боломжтой болгох.

### 3. Хөгжүүлэгч Agent (Core Developer)
- **Үүрэг**: Үндсэн логик, backend үйлдэл болон "хиймэл оюун ухаанаар стикер үүсгэх" зэрэг нарийн функцуудыг хөгжүүлэх.
- **Ашиглах чадварууд (Skills)**:
  - `aider`: Aider арга барилаар (төлөвлөх, хэрэгжүүлэх, шалгах) код бичих.
  - `autoresearch`: Нарийн төвөгтэй алгоритм (AI стикер үүсгэх автоматжуулалт гэх мэт) дээр туршилт хийх.

### 4. Чанар шалгагч ба Тестлэгч Agent (QA & Evaluator)
- **Үүрэг**: Гарсан кодын чанарыг шалгах, алдаа хайх, тест бичих.
- **Ашиглах чадварууд (Skills)**:
  - `polyglot-test-agent`: Unit болон Integration тестүүдийг бичих.
  - `agentic-eval` & `doublecheck`: Үр дүнг эцэслэн шалгах, аюулгүй байдал болон логик алдааг (hallucination) илрүүлэх.

### 5. Баримтжуулагч Agent (Documenter)
- **Үүрэг**: Өөрчлөлтүүдээ баталгаажуулах, README шинэчлэх, Git commit хийх.
- **Ашиглах чадварууд (Skills)**:
  - `git-commit`: Стандартын дагуу commit message үүсгэх.
  - `create-readme`: Төслийн ерөнхий танилцуулгыг бичиж шинэчлэх.

## Хэрхэн ажиллах вэ (Workflow)
1. Хэрэглэгч шинэ даалгавар өгнө. (Жишээ нь: "Нэвтрэх дэлгэц хийе")
2. **Төлөвлөгч** ажлыг ойлгож задална.
3. **UI/UX мэргэжилтэн** харагдах хэсэг (frontend)-ийн кодыг бичнэ.
4. **Хөгжүүлэгч** шаардлагатай логик холболтыг хийнэ.
5. **Тестлэгч** шалгана, **Баримтжуулагч** хадгална.
